JavaFx Scene Builder files into i18 files and edit language properties files

Purpose of this application

When the application is started, you can generate a i18 fxml file from your application unmodified .fxml file. This application will generate also correspond language property file too. And save all new files into another file names. Then at second phase it is possible to edit another language files also.

Sample content of an .fxml file

Normal .fxml file content example rows:

<Button mnemonicParsing="false" text="Open config" />
<Button mnemonicParsing="false" text="Edit config" />

The same content after Compress button pressed:

<Button mnemonicParsing="false" text="%Button5_text" />
<Button mnemonicParsing="false" text="%Button6_text" />

And same content in language properties text area:

Button5_text=Open config
Button6_text=Edit config

When convert and language properties file edit has been done, use these file on following way

Here is a multi language example i18 code when this application is starting:

// FXMLLoader fxmlLoader = new FXMLLoader(JavaFxmlFileConver18Application.class.getResource("javafxmlfileconverti18-view.fxml"));
// This locale can be as selected locale from gui by the user:
Locale locale = new Locale("en", "UK");
ResourceBundle bundle = ResourceBundle.getBundle("com/metait/javafxmlfileI18convert/lang", locale);
// System.out.println("path=" +JavaFxmlFileConver18Application.class.getPackageName().toString());
String strPath = JavaFxmlFileConver18Application.class.getPackageName().replaceAll("\\.","/");
// FXMLLoader fxmlLoader = new FXMLLoader(getClass().getResource("javafxmlfileconverti18-view.fxml"));

// Use here generated .fxml file and indirect also language .propties file:
FXMLLoader fxmlLoader = new FXMLLoader(getClass().getResource("javafxmlfileconverti18-viewtemplate.fxml"), bundle);
// remove xml block from .fxml file: fx:controller="com.metait.javafxplayer.PlayerController"
fxmlLoader.setController(controller);
m_primaryStage = stage;
controller.setPrimaryStage(m_primaryStage);
// Parent loadedroot = fxmlLoader.load();
Scene scene = new Scene(fxmlLoader.load(), 1000, 800);
// JMetro jMetro = new JMetro(scene, Style.LIGHT);
stage.setTitle("JavaFxmlFileI18Convert");
stage.setScene(scene);
stage.show();

Usage

Case 1

As above first chapter, the convert is done at once on following way:

  1. Open a unmodified normal .fxml file
  2. Press Convert button
  3. Save generated modified content of .fml file into another file name

    4. This done for i18 multi language JavaFx application

  4. Save generated content of language properties file into another file name, like: app_name_en.properties or app_name_fi.properties

    5. The stored file name is depend on used language.

  5. After saving both above files, press Edit two language files button
  6. Open on 4 step saved language properties file
  7. Open a new and empty language properties file after above language file names
  8. Copy a content from the first file text area into second empty text area
  9. Translate all sentences on the right side every row after = strike charachter
  10. Save new language content into that new language file

Case 2

At the first case, the convert is done on different steps on following way. That means as in the case 1, but you modify i18 frml app scene builder file, and then convert this later versions. And then add later generated part into a new frml and new language file versions.

  1. Open a all ready i18 modified .fxml file, which have new normal text by example text="text" values enstead of %Button_text variable value.
  2. Press Convert button
  3. Save generated modified content of .fxml file into the same file name
  4. Open an existing language file, which is corresponding current fxml file!

    5. When this is done, the new generated variables will be added the content of just open file

  5. Save generated content of language properties file into the same file name, like: app_name_en.properties or app_name_fi.properties

    6. The stored file name is depend on used language.

  6. After saving both above files, press Edit two language files button etc, like in the case 1
